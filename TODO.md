## Вынести обход AST в свёртку: `foldExpr` / `foldType` / `foldSemType`

Статус: спека согласована на уровне каркаса; два пункта остаются открытыми (см.

### Зачем

Секрет движка — **форма AST**: какие виды узлов есть и где у каждого дети. Сейчас
этот секрет переписан рукописным `switch (kind)` в каждом месте, которое «что-то
делает с выражением/типом» — и внутри движка, и у потребителей:

- движок: `writeLiteral`, `render`/`expr`, `writeType`, `writeQuery`/`step`
  ([writer.ts](src/writer.ts)); `infer` и проверки ([checker.ts](src/checker.ts));
- cli: `describe`/`term`, `display`, `typeLabel`, `controlHints`/`collect`;
- схематор: `structural`/`signatureOfSet`/`collectGroups` (но по `JsonValue`, не по
  AST — см. «Границы»).

Каждый кодирует одну и ту же форму заново. Различается лишь смысл на узле (текст,
метка, контрол, сигнатура), а плита обхода дублируется.

Свойства, которые теряем, держа обход размноженным:

- **Сокрытие формы (Парнас).** Форма AST протекает наружу через каждый внешний
  `switch (kind)`. Добавили вид узла — надо найти и поправить КАЖДУЮ свёртку во всех
  репозиториях.
- **Узкая поверхность во времени (Hyrum / SE@Google).** Должно быть узко («вот тип —
  сверни его»), а вышло широко («вот все виды узлов — обрабатывай каждый»). Цена
  платится на каждом изменении × repos × время.
- **Корректность построением (Дейкстра).** Исчерпанность видов и сама рекурсия в
  детей написаны и протестированы один раз, а не переоткрываются каждым автором.
- **Свёртка/алгебра (Уодлер).** «Как обойти» (рекурсия) отделяется от «что значит»
  (алгебра) — общий рекурсивный скелет параметризуется смыслом.

### Что выносим

Один обход на семейство узлов, параметризованный **алгеброй** — записью «по
обработчику на вид узла». Рекурсию в детей ведёт сам обработчик через переданный
`go` (**открытая рекурсия**) — намеренно, чтобы контекстные потребители работали
(`writer` тянет приоритет, `describe` — режим, `controlHints` спускается только в
`Sub.pred`).

```ts
// Тип-карта над union по `kind`: обработчик ОБЯЗАН быть на КАЖДЫЙ вид.
// Пропущенный вид — ошибка компиляции; новый вид в Expr — ошибка во всех алгебрах,
// пока их не дополнили (исчерпанность by construction, один раз).
type ExprAlgebra<R> = {
  [K in Expr["kind"]]: (node: Extract<Expr, { kind: K }>, go: (e: Expr) => R) => R
};

export function foldExpr<R>(e: Expr, alg: ExprAlgebra<R>): R {
  return (alg[e.kind] as (n: Expr, go: (e: Expr) => R) => R)(e, (c) => foldExpr(c, alg));
}
```

`foldType` (над `TypeExpr`), `foldSemType` (над `SemType`), `foldQuery` — по тому же
образцу со своими алгебрами.

Пример переноса (эталон round-trip): `writeLiteral` → `ExprAlgebra<string>`,
литеральные виды печатают, выраженческие (`Ref`/`Member`/`Apply`/`BinOp`/`UnOp`/
`Underscore`) кидают `WriteError`, как сейчас.

### Границы (что НЕ делаем)

- **`SemType` и `TypeExpr` не объединять** — это разные домены (семантика vs
  синтаксис), похожие, но не совпадающие; каждому свой fold.
- **Обход `JsonValue` в схематоре не тянуть в движок** — движку про JSON знать
  незачем. Тот же *паттерн* применяется там **локально**, носитель другой.
- **Инварианты сохранить:** round-trip `parseX(writeX(ast)) = ast` (тесты writer),
  поведение `WriteError` на не-литерале, минимальные скобки по иерархии приоритетов.

## Повторный поиск похожих паттернов (повторяемо, не разово)

демо и потребителей, поэтому поиск надо завести как **повторяющийся проход**, а не
разовый акт.

  тот же класс дублей: рукописные обходы AST (`switch (kind)` + рекурсия), магические
  строки системных доменов (`Число`/`Строка`/…, канон уже в [checker.ts:39](src/checker.ts#L39),
  не экспортируется), параллельные обходы одной структуры (триплет схематора), пары
  почти-близнецов (cli `readField`/`readFieldEdit`, `addOne`/`editOne`).
- [ ] Искать **до иссякания** (loop-until-dry): прекращаем, когда проход не даёт новых
  находок, а не после первой.
- [ ] Критерий «куда выносить»:
  - про AST / типы / нотацию → **engine**;
  - общий механизм вне домена нотации → **новая библиотека** (см. ниже);
  - специфичное одному демо → остаётся **локально**.
